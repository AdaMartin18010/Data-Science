### 2. 标题层级规范

**一级标题**：使用中文数字（一、二、三...）

```markdown
## 一、一级标题名称

## 二、一级标题名称

## 三、一级标题名称
```

**二级标题**：使用数字编号（1.1, 1.2, 2.1...）

```markdown
### 1.1 二级标题名称

### 1.2 二级标题名称

### 2.1 二级标题名称
```

**三级标题**：使用数字编号（1.1.1, 1.1.2...）

```markdown
#### 1.1.1 三级标题名称

#### 1.1.2 三级标题名称
```

**四级标题**：使用数字编号（1.1.1.1...）

```markdown

```

- **一级章节**：一、二、三、四、五、六、七、八、九、十...
- **二级章节**：1.1, 1.2, 1.3... 2.1, 2.2...

## 📝 内容格式规范

### 1. 代码块

```markdown
    **语言标识**：

    ```sql
    -- SQL代码示例
    SELECT * FROM users;
    ```

    ```c
    // C代码示例
    sqlite3_open("app.db", &db);
    ```

    ```python
    # Python代码示例
    conn = sqlite3.connect('app.db')
    ```

```

### 2. 表格

```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 数据1 | 数据2 | 数据3 |
```

### 3. 列表

**有序列表**：

```markdown
1. 第一项
2. 第二项
3. 第三项
```

**无序列表**：

```markdown
- 项目1
- 项目2
- 项目3
```

**带标记的列表**：

```markdown
- ✅ 已完成
- 📋 待开始
- 🔄 进行中
- ❌ 不推荐
```

### 4. 强调和标记

```markdown
**粗体**：重要概念
*斜体*：强调
`代码`：代码片段
> 引用：重要说明
```

## 📐 目录生成规则

### 1. 自动目录

对于长文档（>500行），必须包含目录：

```markdown
## 📑 目录

- [一、一级标题](#一、一级标题)
  - [1.1 二级标题](#11-二级标题)
  - [1.2 二级标题](#12-二级标题)
- [二、一级标题](#二、一级标题)
```

### 2. 锚点规则

- 中文标题：使用拼音或简化形式
- 数字标题：直接使用数字
- 特殊字符：使用连字符替换空格

示例：

- `## 一、数据类型系统` → `#一、数据类型系统`
- `### 1.1 存储类` → `#11-存储类`
- `### 2.1 类型亲和性` → `#21-类型亲和性`
