## ❌ 不适用场景

### 高并发写入场景

**问题**：数据库级写锁，无法水平扩展

**替代方案**：

- PostgreSQL/MySQL
- Redis + 关系型数据库组合

**相关文档**：

- [04.02 不适用场景论证](./04-应用场景/04.02-不适用场景论证.md)
- [05.02 选型决策框架](./05-对比选型/05.02-选型决策框架.md)

---

### 海量数据分析（OLAP）

**问题**：无分区表、无并行查询能力

**替代方案**：

- PostgreSQL（分析型查询）
- ClickHouse（列式存储）
- 数据仓库解决方案

**相关文档**：

- [04.02 不适用场景论证](./04-应用场景/04.02-不适用场景论证.md)
- [05.01 多维知识矩阵](./05-对比选型/05.01-多维知识矩阵.md)

---

### 多用户网络服务

**问题**：无内置用户权限管理

**替代方案**：

- PostgreSQL/MySQL（C/S架构）
- 通过连接池和权限隔离保障安全

## 🔍 场景选择决策树

```text
开始
  │
  ├─ 是否需要高并发写入（>1000 TPS）？
  │   ├─ 是 → 使用PostgreSQL/MySQL
  │   └─ 否 → 继续
  │
  ├─ 数据量是否 > 100GB？
  │   ├─ 是 → 考虑分片或C/S数据库
  │   └─ 否 → 继续
  │
  ├─ 是否需要多用户权限管理？
  │   ├─ 是 → 使用C/S数据库
  │   └─ 否 → 继续
  │
  ├─ 应用是否嵌入式/单机？
  │   ├─ 是 → SQLite3首选 ✅
  │   └─ 否 → 继续
  │
  └─ 是否测试/临时数据？
      ├─ 是 → SQLite :memory:模式 ✅
      └─ 否 → SQLite文件模式 ✅
```
