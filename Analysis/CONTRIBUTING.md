# 贡献者指南

## 欢迎贡献

感谢您对数据科学知识库项目的关注！我们欢迎所有形式的贡献，包括但不限于：

- 内容改进和扩展
- 错误修复
- 新功能开发
- 文档完善
- 翻译工作
- 代码优化

## 贡献流程

### 1. 准备工作

- Fork 本仓库
- 克隆到本地：`git clone https://github.com/your-username/Data-Science.git`
- 创建新分支：`git checkout -b feature/your-feature-name`

### 2. 开发规范

#### 2.1 文档规范

- 遵循 [国际化Wiki标准与知识规范对齐指南](国际化Wiki标准与知识规范对齐指南.md)
- 使用 Markdown 格式
- 包含完整的元数据（frontmatter）
- 提供中英文双语定义（核心概念）

#### 2.2 代码规范

- Python 代码遵循 PEP 8
- SQL 代码遵循 PostgreSQL 规范
- 添加必要的注释和文档字符串

#### 2.3 提交规范

提交信息格式：
```
类型(范围): 简短描述

详细描述（可选）

相关Issue: #123
```

类型包括：
- `feat`: 新功能
- `fix`: 错误修复
- `docs`: 文档更新
- `style`: 代码格式
- `refactor`: 重构
- `test`: 测试
- `chore`: 构建/工具

### 3. 质量检查

提交前请运行：

```bash
# 质量检查
python quality_checker.py . --format json --output quality_report.json

# 链接检查
python link_check.py

# 格式检查
python -m markdownlint *.md
```

### 4. 提交 Pull Request

1. 确保所有检查通过
2. 更新相关文档
3. 添加测试（如适用）
4. 提交 PR 并描述变更内容

### 5. 代码审查

- PR 需要至少一位维护者审查
- 根据反馈进行修改
- 审查通过后合并

## 翻译贡献

### 翻译优先级

1. **Critical**: 核心概念定义、数学公式、API文档
2. **High**: 理论背景、算法描述、最佳实践
3. **Medium**: 历史背景、相关研究、扩展阅读
4. **Low**: 个人观点、实验性内容

### 翻译规范

- 保持术语一致性
- 使用标准术语表
- 保留代码和公式原文
- 提供双语对照
