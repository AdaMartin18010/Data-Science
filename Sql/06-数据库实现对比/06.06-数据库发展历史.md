## 一、概述

### 1.1 数据库发展时间线

```text
┌─────────────────────────────────────────────────────────────────────────┐
│                       关系数据库发展时间线                                │
└─────────────────────────────────────────────────────────────────────────┘

1970 ─────────────────────────────────────────────────────────────────►
  │
  ├─ 1970: Codd发表关系模型论文 "A Relational Model of Data"
  │
  ├─ 1974: IBM System R项目启动
  │
  ├─ 1977: Oracle成立（原名SDL）
  │
  ├─ 1979: Oracle V2发布（首个商业关系数据库）
  │
1980 ─────────────────────────────────────────────────────────────────►
  │
  ├─ 1986: SQL成为ANSI/ISO标准 (SQL-86)
  │
  ├─ 1986: POSTGRES项目启动 (UC Berkeley)
  │
  ├─ 1988: SQL Server 1.0发布 (Sybase/Microsoft)
  │
  ├─ 1989: SQL-89标准发布
  │
1990 ─────────────────────────────────────────────────────────────────►
  │
  ├─ 1992: SQL-92标准发布
  │
  ├─ 1995: MySQL 1.0发布
  │
  ├─ 1996: PostgreSQL 6.0发布（重命名）
  │
  ├─ 1999: SQL:1999标准发布（对象关系扩展）
  │
2000 ─────────────────────────────────────────────────────────────────►
  │
  ├─ 2000: SQLite 1.0发布
  │
  ├─ 2003: SQL:2003标准发布（窗口函数）
  │
  ├─ 2005: PostgreSQL 8.0发布（Windows原生支持）
  │
  ├─ 2008: MySQL被Sun收购
  │
2010 ─────────────────────────────────────────────────────────────────►
  │
  ├─ 2010: Oracle收购Sun（MySQL）
  │
  ├─ 2011: SQL:2011标准发布（时态数据）
  │
  ├─ 2012: PostgreSQL 9.2发布（JSON支持）
  │
  ├─ 2016: SQL:2016标准发布（JSON）
  │
2020 ─────────────────────────────────────────────────────────────────►
  │
  ├─ 2021: PostgreSQL 14发布
  │
  ├─ 2023: SQL:2023标准发布（图查询）
  │
  ├─ 2024: PostgreSQL 17发布（向量支持增强）
  │
  └─ 2025: AI/向量数据库时代
```

### 1.2 关系数据库发展阶段

```text
阶段1: 理论奠基期 (1970-1979)
─────────────────────────────
- Codd关系模型
- System R / INGRES
- SQL语言诞生

阶段2: 商业化时期 (1980-1999)
─────────────────────────────
- Oracle, DB2商业化
- SQL标准化
- PostgreSQL/MySQL开源

阶段3: 开源繁荣期 (2000-2015)
─────────────────────────────
- LAMP栈流行
- NoSQL运动
- 云数据库兴起

阶段4: 多模时代 (2015-2023)
─────────────────────────────
- JSON/文档支持
- 时序/图数据库
- 分布式数据库

阶段5: AI融合时代 (2024-)
─────────────────────────────
- 向量数据库
- Text-to-SQL
- 智能优化器
```

## 二、PostgreSQL发展历史

### 2.1 起源与早期发展

```text
┌─────────────────────────────────────────────────────────────────────────┐
│                       PostgreSQL发展历程                                 │
└─────────────────────────────────────────────────────────────────────────┘

项目起源:
─────────
1977: Michael Stonebraker在UC Berkeley创建INGRES项目
      - 第一代关系数据库系统
      - 使用QUEL查询语言

1986: POSTGRES项目启动（POST-INGRES）
      - 目标：超越INGRES的局限性
      - 引入对象关系模型
      - 支持复杂数据类型

1995: Postgres95发布
      - 添加SQL语言支持
      - 开源社区开始维护

1996: 重命名为PostgreSQL
      - 强调SQL支持
      - 版本号重置为6.0

创始团队:
─────────
- Michael Stonebraker (项目领导)
- Lawrence Rowe (共同领导)
- 加州大学伯克利分校研究团队
```

### 2.2 重要版本里程碑

| 版本 | 年份 | 重要特性 | 影响 |
|------|------|---------|------|
| 6.0 | 1996 | 首个PostgreSQL版本 | 奠定社区基础 |
| 7.0 | 2000 | WAL日志、外键 | 企业级可靠性 |
| 7.4 | 2003 | 性能优化、子查询改进 | 性能突破 |
| 8.0 | 2005 | Windows原生、表空间、PITR | 跨平台支持 |
| 8.4 | 2009 | 窗口函数、CTE | SQL:2003支持 |
| 9.0 | 2010 | 流复制、热备份 | 高可用支持 |
| 9.2 | 2012 | JSON支持、索引改进 | NoSQL特性 |
| 9.4 | 2014 | JSONB、逻辑复制 | 文档数据库能力 |
| 10 | 2017 | 声明式分区、逻辑复制 | 大数据支持 |
| 12 | 2019 | 性能优化、CTE改进 | 分析能力增强 |
| 14 | 2021 | 性能提升、连接管理 | 高并发优化 |
| 15 | 2022 | MERGE语句、压缩 | SQL标准增强 |
| 16 | 2023 | 并行查询改进 | 分析性能 |
| 17 | 2024 | 向量索引增强 | AI集成 |

### 2.3 PostgreSQL技术演进

```text
┌─────────────────────────────────────────────────────────────────────────┐
│                    PostgreSQL技术演进图                                  │
└─────────────────────────────────────────────────────────────────────────┘

1996-2000: 基础建设
    ┌─────────────┐
    │ SQL支持     │ → ACID事务 → 外键约束
    └─────────────┘

2000-2010: 企业级特性
    ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
    │ WAL日志     │ ──► │ PITR恢复     │ ──► │ 流复制      │
    └─────────────┘     └─────────────┘     └─────────────┘

2010-2020: 扩展能力
    ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
    │ JSON/JSONB  │ ──► │ 扩展系统     │ ──► │ 分区表      │
    └─────────────┘     └─────────────┘     └─────────────┘

2020-2025: AI/向量时代
    ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
    │ pgvector    │ ──► │ 混合检索     │ ──► │ RAG集成     │
    └─────────────┘     └─────────────┘     └─────────────┘

关键扩展生态:
─────────────
PostGIS (地理空间) → TimescaleDB (时序) → pgvector (向量)
    ↓                      ↓                    ↓
 GIS应用              IoT/监控             AI/语义检索
```

## 三、MySQL发展历史

### 3.1 起源与早期发展

```text
┌─────────────────────────────────────────────────────────────────────────┐
│                        MySQL发展历程                                     │
└─────────────────────────────────────────────────────────────────────────┘

项目起源:
─────────
1994: MySQL AB公司成立（瑞典）
      - 创始人：Michael Widenius, David Axmark
      - 名字来源：Michael的女儿"My"

1995: MySQL 1.0内部发布
      - 简单、快速的数据库
      - 专注于Web应用

1996: MySQL 3.19公开发布
      - 支持基本SQL
      - 高性能读取

设计哲学:
─────────
- 速度优先
- 易于使用
- 可靠性
- 开源+商业双授权
```

### 3.2 重要版本里程碑

| 版本 | 年份 | 重要特性 | 影响 |
|------|------|---------|------|

| 4.0 | 2003 | 查询缓存、嵌套查询 | 性能提升 |
| 4.1 | 2004 | 子查询、UTF-8 | 国际化 |
| 5.0 | 2005 | 存储过程、触发器、视图 | 企业级特性 |
| 5.1 | 2008 | 分区、事件调度器 | 大数据支持 |
| 5.5 | 2010 | InnoDB默认、性能优化 | 事务默认 |
| 5.6 | 2013 | 全文索引(InnoDB)、优化器 | 分析能力 |
| 5.7 | 2015 | JSON支持、GIS | 多模支持 |
| 8.0 | 2018 | 窗口函数、CTE、角色 | 现代SQL |
| 8.0.32+ | 2023 | HeatWave向量 | AI集成 |

┌─────────────────────────────────────────────────────────────────────────┐

## 四、SQLite发展历史

### 4.1 起源与设计哲学

```text
┌─────────────────────────────────────────────────────────────────────────┐
│                        SQLite发展历程                                    │
└─────────────────────────────────────────────────────────────────────────┘

项目起源:
─────────
2000年8月: D. Richard Hipp创建SQLite
           - 原因：为美国海军导弹驱逐舰项目开发
           - 需求：无需服务器的嵌入式数据库

设计目标:
─────────
1. 零配置 (Zero-configuration)
2. 无服务器 (Serverless)
3. 单文件 (Single-file)
4. 事务安全 (Transactional)
5. 小巧 (Small footprint)

授权模式:
─────────
公有领域 (Public Domain) - 无任何限制
```

### 4.2 重要版本里程碑

| 版本 | 年份 | 重要特性 | 影响 |
|------|------|---------|------|
| 1.0 | 2000 | 首个公开版本 | 嵌入式数据库诞生 |
| 2.0 | 2001 | 改进SQL支持 | 功能增强 |
| 3.0 | 2004 | 类型亲和性、UTF-8 | 现代SQL基础 |
| 3.6 | 2009 | 外键支持、WAL模式 | 并发性能 |
| 3.7 | 2010 | WAL模式改进 | 高并发 |
| 3.8 | 2013 | CTE、窗口函数预备 | 分析能力 |
| 3.25 | 2018 | 窗口函数 | 现代SQL |
| 3.35 | 2021 | 数学函数、ALTER列 | DDL增强 |
| 3.38 | 2022 | JSON改进 | 文档支持 |
| 3.45 | 2024 | JSON5支持 | 现代JSON |
