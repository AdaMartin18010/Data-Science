## 一、框架概述

### 1.1 设计目标

```text
┌─────────────────────────────────────────────────────────────────────────┐
│                        统一概念定义框架目标                              │
└─────────────────────────────────────────────────────────────────────────┘

目标1: 因果性完整
─────────────────
- 每个概念必须标注其前因(causes)和后果(effects)
- 概念间的推导关系必须有形式化证明或论证
- 因果链无断裂、无跳跃

目标2: 关联性显式
─────────────────
- 每个概念必须标注与其他概念的关系类型
- 关系类型包括：is-a, has-a, depends-on, equivalent-to等
- 支持概念间的双向导航

目标3: 形式化严格
─────────────────
- 提供形式化定义（符号、公式）
- 提供公理系统
- 提供定理和证明

目标4: 对标权威
─────────────────
- 对标Wikipedia概念条目结构
- 对标国际大学课程内容
- 对标学术论文规范
```

### 1.2 对标标准

| 标准来源 | 核心要素 | 本框架对应 |
|---------|---------|-----------|
| **Wikipedia** | 定义、历史、属性、分类、关系、应用 | 完整涵盖 |
| **Stanford CS145** | 形式化定义、证明、应用 | 形式化层次 |

## 三、关系类型定义

### 3.1 概念间关系类型

```text
┌─────────────────────────────────────────────────────────────────────────┐
│                        概念关系类型体系                                  │
└─────────────────────────────────────────────────────────────────────────┘

1. 层次关系 (Hierarchical)
   ├── is-a (泛化/特化)
   │   例：关系代数 is-a 代数系统
   │
   └── part-of / has-a (组成)
       例：属性 part-of 关系

2. 依赖关系 (Dependency)
   ├── depends-on (前置依赖)
   │   例：SQL完备性 depends-on 关系代数完备性
   │
   ├── enables (使能)
   │   例：索引 enables 快速查询
   │
   └── requires (需要)
       例：ACID requires 日志系统

3. 等价关系 (Equivalence)
   ├── equivalent-to (等价)
   │   例：关系代数 equivalent-to 关系演算 (在表达能力上)
   │
   └── isomorphic-to (同构)
       例：关系 isomorphic-to 表

4. 对立关系 (Opposition)
   ├── contradicts (矛盾)
   │   例：READ UNCOMMITTED contradicts 强一致性
   │
   └── trade-off (权衡)
       例：一致性 trade-off 可用性 (CAP)

5. 因果关系 (Causality)
   ├── causes (导致)
   │   例：数据冗余 causes 更新异常
   │
   └── prevents (阻止)
       例：范式化 prevents 数据冗余

6. 映射关系 (Mapping)
   ├── implements (实现)
   │   例：SELECT implements 投影操作
   │
   └── abstracts (抽象)
       例：关系代数 abstracts SQL语义
```

### 3.2 关系表示规范

```text
关系表示格式：
─────────────
【主体概念】 ──[关系类型: 说明]──→ 【客体概念】

示例：
─────
关系模型 ──[is-a: 数据模型类型]──→ 数据模型
属性 ──[part-of: 组成元素]──→ 关系
SQL ──[implements: 语言实现]──→ 关系代数
部分依赖 ──[causes: 导致问题]──→ 数据冗余
2NF ──[prevents: 消除问题]──→ 部分依赖
```

### 4.2 推理规则格式

```text
推理规则模板：
─────────────

Rule: [规则名称]
────────────────────────────────────────────────────────

Premises (前提):
  P1: 前提条件1
  P2: 前提条件2
  ...

Conclusion (结论):
  C: 结论

Proof (证明):
  1. 由P1，得...
  2. 由P2和步骤1，得...
  3. 因此，C成立

Confidence (置信度): 必然/高/中/低

Application (应用):
  当观察到P1和P2时，可推断C

Counter-example (反例):
  如果存在反例，说明条件

References (依据):
  - 定理X
  - 公理Y
```

**示例：选择下推规则**:

```text
Rule: Selection Pushdown (选择下推)
────────────────────────────────────────────────────────

Premises:
  P1: σ_p(R ⋈ S) 需要计算
  P2: 谓词p仅涉及关系R的属性
  P3: R ⋈ S 是自然连接或等值连接

Conclusion:
  C: σ_p(R ⋈ S) ≡ σ_p(R) ⋈ S

Proof:
  1. 设 t ∈ σ_p(R ⋈ S)
  2. 则 t ∈ R ⋈ S 且 p(t) = true
  3. 由连接定义，∃r∈R, s∈S: t = r ⋈ s
  4. 因p仅涉及R的属性，p(t) = p(r)
  5. 故 r ∈ σ_p(R)
  6. 因此 t = r ⋈ s ∈ σ_p(R) ⋈ S
  7. 反向同理可证
  8. 故 σ_p(R ⋈ S) ≡ σ_p(R) ⋈ S

Confidence: 必然 (数学证明)

Application:
  查询优化器可将过滤条件下推到连接之前执行，
  减少连接的数据量，提高查询性能

References:
  - 关系代数等价性定理
  - 查询优化理论
```

## 六、核心概念定义示例

### 6.1 关系(Relation)

---

#### 【关系】(Relation)

> **所属模块**：01-理论基础
> **形式化层次**：L4-可验证
> **前置概念**：[集合](链接), [笛卡尔积](链接), [域](链接)
> **后续概念**：[关系模式](链接), [关系代数](链接), [表](链接)

---

##### 1. 形式化定义

**1.1 符号定义**:

- D₁, D₂, ..., Dₙ：域（属性的取值范围）
- R：关系
- t：元组
- A₁, A₂, ..., Aₙ：属性名

**1.2 数学定义**:

```text
定义：关系R是域D₁, D₂, ..., Dₙ的笛卡尔积的有限子集

R ⊆ D₁ × D₂ × ... × Dₙ

其中：
- D₁ × D₂ × ... × Dₙ = {(d₁, d₂, ..., dₙ) | dᵢ ∈ Dᵢ}
- R是有限集合
- |R| 表示关系的基数（元组数量）
```

**1.3 前置条件**:

- 每个Dᵢ是非空集合
- n ≥ 1（至少有一个属性）

**1.4 等价定义**:

- 定义1：关系是元组的集合
- 定义2：关系是从属性到值的函数的集合
- 定义3：关系是二维表的数学抽象

##### 2. 属性

**2.1 必要属性**:

| 属性 | 形式化表示 | 说明 |
|------|-----------|------|
| 无序性 | {t₁, t₂} = {t₂, t₁} | 元组没有固定顺序 |
| 无重复性 | t₁ = t₂ ⟹ 同一元组 | 不允许重复元组 |
| 原子性 | ∀t∈R, ∀i: t[Aᵢ] ∈ Dᵢ | 每个属性值是原子的 |

**2.2 衍生属性**:

### 6.2 选择操作(Selection)

---

#### 【选择操作】(Selection, σ)

> **所属模块**：01-理论基础 / 03-形式化模型
> **形式化层次**：L4-可验证
> **前置概念**：[关系](链接), [谓词](链接)
> **后续概念**：[WHERE子句](链接), [过滤算子](链接)

---

##### 1. 形式化定义

**1.1 符号定义**:

- σ：选择操作符
- p：选择谓词（布尔表达式）
- R：输入关系
- σ_p(R)：选择操作结果

**1.2 数学定义**:

```text
σ_p(R) = {t | t ∈ R ∧ p(t) = true}

其中p(t)表示在元组t上评估谓词p的结果
```

**1.3 等价定义**:

- 集合论：选择是满足条件的元组的集合
- 逻辑：选择是对关系应用一阶谓词过滤

##### 2. 属性

**2.1 代数性质**:

| 性质 | 形式化表示 | 证明状态 |
|------|-----------|---------|
| 幂等性 | σ_p(σ_p(R)) = σ_p(R) | ✅已证明 |
| 交换性 | σ_p(σ_q(R)) = σ_q(σ_p(R)) | ✅已证明 |
| 合并性 | σ_p(σ_q(R)) = σ_{p∧q}(R) | ✅已证明 |
| 分解性 | σ_{p∧q}(R) = σ_p(σ_q(R)) | ✅已证明 |

##### 3. 定理与证明

**定理：选择下推**:

```text
定理：σ_p(R ⋈ S) ≡ σ_p(R) ⋈ S，当p仅涉及R的属性

证明：
设 t ∈ σ_p(R ⋈ S)
⟺ t ∈ R ⋈ S ∧ p(t) = true
⟺ ∃r∈R, s∈S: t = r⋈s ∧ p(r⋈s) = true
⟺ ∃r∈R, s∈S: t = r⋈s ∧ p(r) = true  (因p仅涉及R)
⟺ ∃r∈σ_p(R), s∈S: t = r⋈s
⟺ t ∈ σ_p(R) ⋈ S

Q.E.D.
```

##### 4. 应用映射

| 理论 | SQL | 执行算子 |
|------|-----|---------|
| σ_p(R) | SELECT * FROM R WHERE p | Filter |
| σ_{A>10}(R) | SELECT * FROM R WHERE A > 10 | Filter(A > 10) |

```sql
-- 理论：σ_{age>20}(Student)
-- SQL实现：
SELECT * FROM Student WHERE age > 20;
```

### 6.3 函数依赖(Functional Dependency)

---

#### 【函数依赖】(Functional Dependency)

> **所属模块**：01-理论基础
> **形式化层次**：L4-可验证
> **前置概念**：[关系](链接), [属性](链接), [函数](链接)
> **后续概念**：[范式](链接), [规范化](链接), [Armstrong公理](链接)

---

##### 1. 形式化定义

**1.1 符号定义**:

- X, Y：属性集合
- X → Y：X函数决定Y
- R：关系模式
- r：关系实例

**1.2 数学定义**:

```text
定义：X → Y 在关系r上成立，当且仅当：

∀t₁, t₂ ∈ r: t₁[X] = t₂[X] ⟹ t₁[Y] = t₂[Y]

即：如果两个元组在X上的值相同，则它们在Y上的值也相同
```

##### 2. 公理系统 (Armstrong公理)

```text
Armstrong公理系统：

A1 (自反律): 若Y ⊆ X，则X → Y
A2 (增广律): 若X → Y，则XZ → YZ
A3 (传递律): 若X → Y 且 Y → Z，则X → Z

推导规则：

R1 (合并律): 若X → Y 且 X → Z，则X → YZ
R2 (分解律): 若X → YZ，则X → Y 且 X → Z
R3 (伪传递律): 若X → Y 且 WY → Z，则WX → Z
```

##### 3. 因果链

```text
函数依赖因果链：
─────────────────

函数依赖分析
    │
    ├──[存在部分依赖]──→ 数据冗余 ──→ 更新异常
    │       │
    │       └──[解决]──→ 分解到2NF
    │
    ├──[存在传递依赖]──→ 数据冗余 ──→ 插入/删除异常
    │       │
    │       └──[解决]──→ 分解到3NF
    │
    └──[存在非候选键依赖]──→ 异常
            │
            └──[解决]──→ 分解到BCNF

推理规则：
─────────
Rule: 范式选择
If: 存在部分函数依赖
Then: 需要分解到2NF以上
Proof: 部分依赖导致数据冗余，冗余导致更新异常
```
