## 🎯 贡献方式

### 1. 文档完善

- 修正错别字和语法错误
- 改进文档表述
- 补充示例代码
- 添加参考资源

### 2. 内容扩展

- 扩展现有文档内容
- 添加新的设计模式
- 补充真实案例
- 添加最佳实践

### 3. 工具开发

- 开发自动化检查工具
- 创建文档生成脚本
- 实现质量评估工具

### 4. 问题反馈

- 报告文档错误
- 提出改进建议
- 讨论技术问题

## 📝 贡献流程

### 步骤1：了解项目

1. 阅读[项目README](../README.md)
2. 浏览[INDEX主题索引](./INDEX.md)
3. 查看[文档格式规范](./03-规范模板/文档格式规范.md)

### 步骤2：选择任务

1. 查看[待办任务清单](./待办任务清单.md)
2. 选择适合自己的任务
3. 确认任务未被他人认领

### 步骤3：创建分支

```bash
# 从main分支创建新分支
git checkout -b feature/your-feature-name

# 或修复bug
git checkout -b fix/bug-description
```

### 步骤4：编写内容

1. 遵循[文档格式规范](./03-规范模板/文档格式规范.md)
2. 使用清晰的中文表述
3. 添加必要的代码示例
4. 更新相关索引文档

### 步骤5：提交更改

```bash
# 添加更改
git add .

# 提交（使用清晰的提交信息）
git commit -m "feat: 添加向量数据库设计案例"

# 推送
git push origin feature/your-feature-name
```

### 步骤6：创建PR

1. 在GitHub创建Pull Request
2. 填写PR描述模板
3. 等待代码审查
4. 根据反馈修改

## 📋 提交规范

### 提交信息格式

```text
<type>: <subject>

<body>

<footer>
```

### Type类型

| Type | 说明 |
|------|------|
| feat | 新功能、新文档 |
| fix | 修复错误 |
| docs | 文档更新 |
| style | 格式调整 |
| refactor | 重构 |
| chore | 其他更改 |

### 示例

```text
feat: 添加07.41-微服务数据库设计案例

- 添加微服务架构数据库设计实践
- 包含Saga模式实现示例
- 添加事件溯源设计方案

Related: #123
```

## ✅ 质量要求

### 文档要求

- 遵循[文档格式规范](./03-规范模板/文档格式规范.md)
- 内容准确、表述清晰
- 代码示例可运行
- 链接有效

### 代码要求

- 代码风格一致
- 添加必要注释
- 包含输出示例

### 审查要点

- 内容正确性
- 格式规范性
- 链接有效性
- 示例完整性
